You’re correct. Fix it now on the frontend repo (levqor-site). Pick the path that matches your setup.

Path A — Separate frontend repo on GitHub (Vercel deploys from it)

1. Clone and check



git clone https://github.com/<ORG>/<levqor-site>.git
cd levqor-site
git remote -v
git status

2. Add public files



mkdir -p public
cat > public/robots.txt <<'TXT'
User-agent: *
Allow: /
Sitemap: https://levqor.ai/sitemap.xml
TXT

cat > public/humans.txt <<'TXT'
Team: Levqor Engineering
Site: https://levqor.ai
TXT

cat > public/security.txt <<'TXT'
Contact: mailto:security@levqor.ai
Policy: https://levqor.ai/security
Preferred-Languages: en
TXT

3. Commit and push



git add public/robots.txt public/humans.txt public/security.txt
git commit -m "chore: add robots.txt, humans.txt, security.txt to frontend"
git push origin main

4. Verify Vercel production deploy



Vercel → Project levqor-site → Deployments → latest → Promote/Assign to Production if not auto.

Confirm on wire:


curl -I https://levqor.ai | grep -iE 'cache-control|age|x-vercel-cache'
curl -s https://levqor.ai/robots.txt
curl -s https://levqor.ai/humans.txt
curl -s https://levqor.ai/.well-known/security.txt || curl -s https://levqor.ai/security.txt

Path B — Monorepo (frontend in a subfolder of the backend repo)

1. Ensure Vercel project uses a subdirectory



Vercel → Project levqor-site → Settings → Root Directory = levqor-site (or your subfolder).

If not set, set it and redeploy.


2. Add files inside the frontend subfolder



cd /path/to/repo/levqor-site
mkdir -p public
# same three cat commands as Path A
git add public/robots.txt public/humans.txt public/security.txt
git commit -m "chore(frontend): add robots/humans/security"
git push origin main

3. Verify (same curl checks as above).
