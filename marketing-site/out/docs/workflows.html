Workflow Engine API

The Workflow Engine provides live automation monitoring and control through a RESTful API.

Endpoints

GET /api/workflows

List all workflows with current stats.

Response Fields

  • id (string) - Unique workflow identifier
  • name (string) - Human-readable workflow name
  • enabled (boolean) - Whether workflow is enabled
  • last_run (number) - Unix timestamp of last execution
  • status (string) - One of: ok, degraded, fail
  • avg_latency_ms (number) - Average execution time in milliseconds
  • success_7d (number) - Successful runs in last 7 days
  • fail_7d (number) - Failed runs in last 7 days

Example Response

[
  {
    "id": "self_heal",
    "name": "Self-Heal Cycle",
    "enabled": true,
    "last_run": 1729900000,
    "status": "ok",
    "avg_latency_ms": 245.5,
    "success_7d": 168,
    "fail_7d": 0
  },
  {
    "id": "health_monitor",
    "name": "Health Monitor",
    "enabled": true,
    "last_run": 1729899700,
    "status": "ok",
    "avg_latency_ms": 120.3,
    "success_7d": 504,
    "fail_7d": 2
  }
]

GET /api/workflows/<id>/runs

Get the last 50 execution runs for a specific workflow.

Parameters

  • id (path) - Workflow ID

Response Fields

  • ts (number) - Unix timestamp of run
  • duration_ms (number) - Execution time in milliseconds
  • status (string) - Run status: ok or fail
  • note (string) - Additional context or error message

Example Response

[
  {
    "ts": 1729900000,
    "duration_ms": 250,
    "status": "ok",
    "note": "Self-heal cycle #42"
  },
  {
    "ts": 1729896400,
    "duration_ms": 220,
    "status": "ok",
    "note": "Self-heal cycle #41"
  }
]

POST /api/workflows/<id>/toggle

Enable or disable a workflow. Toggles the current state.

Parameters

  • id (path) - Workflow ID

Response Fields

  • ok (boolean) - Whether operation succeeded
  • workflow_id (string) - ID of toggled workflow
  • enabled (boolean) - New enabled state

Example Response

{
  "ok": true,
  "workflow_id": "self_heal",
  "enabled": false
}

Rate Limits

All workflow API endpoints are rate-limited to 5 requests per second per IP address. Exceeding this limit returns HTTP 429 with error message.

Error Responses

404 Not Found

{
  "ok": false,
  "error": "Workflow not found"
}

429 Rate Limit Exceeded

{
  "ok": false,
  "error": "Rate limit exceeded"
}

500 Internal Server Error

{
  "ok": false,
  "error": "Error message here"
}

Data Sources

Workflow data is aggregated from multiple log files:

  • logs/self_heal.ndjson - Self-healing cycle execution logs
  • logs/auto_rollback.ndjson - Automatic rollback logs
  • exports/uptime_*.json - Health monitoring uptime data

If log files are missing or empty, the API returns mock data to keep endpoints stable.
